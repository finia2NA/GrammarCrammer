# GrammarCrammer — Project Description & Progress

## Overview
AI-assisted grammar studying app, flashcard style. React Native (Expo, web-first).
Core loop: user names a grammar topic → Claude generates an explanation + flashcards →
user practices translating English sentences → Claude judges answers in real time.

---

## Full Feature Description

### Step 1 — Onboarding ✅ DONE
On first launch, the app checks local storage for a Claude API key.
If none is found, it shows an onboarding flow: a card carousel with placeholder text,
ending with a card that asks for the Claude API key.
When submitted, the app makes a test request to verify the key works.
On success the key is saved and the home page is shown.

### Step 2 — Home Screen ✅ Done
A text box with greyed-out placeholder text: "What shall we study today".
To the right: a dropdown for target language, a toggle for mode (練習 / 習得),
and a dropdown for the number of cards to generate.
A button starts the study session.

### Step 3 — Study Session ✅ Done

#### Explanation phase
- **習得 mode**: A full-screen card explains the grammar topic (generated by Claude Sonnet).
  The user dismisses it; it slides to the side and stays visible as a reference panel.
- **練習 mode**: The full-screen explanation is skipped; the explanation goes to the side
  reference panel from the very beginning (still generated in the background).

#### Flashcard loop
- Claude Haiku generates the requested number of cards for the topic.
  Cards are always English → target language, and each example should be unambiguous
  (i.e. a sentence where the specific grammar point is clearly the best choice).
- Cards are shuffled randomly before the session begins.
- The user sees the English sentence and must type the target-language version.
- Pressing Enter submits: Claude Haiku judges whether the answer is correct
  (uses the right grammar and sounds natural).
  - **Correct**: confirmation shown → Enter → card removed, next card shown.
  - **Wrong**: Claude Sonnet explains why it was rejected and what would be better →
    user confirms → card moved to end of stack.
- Session ends when the stack is empty.

## Tech Stack
- **Framework**: Expo 54 (React Native + Web)
- **Language**: TypeScript
- **Navigation**: Expo Router (file-based)
- **Storage**: @react-native-async-storage/async-storage
- **AI**: Direct fetch to Anthropic API (user's own key, no backend)
- **Styling**: NativeWind v4 (Tailwind CSS)
- **Models**: Claude Sonnet 4.6 (explanations), Claude Haiku 4.5 (cards + judgment)
